# Task5. GraphQL для client-info

В файле `schema.graphql` лежит схема, эквивалентная REST API сервиса client-info из контракта Swagger.

Запрос `client(id)` возвращает клиента, а вложенные поля `documents` и `relatives` заменяют отдельные ресурсы REST. Потребитель сам выбирает нужные поля, поэтому пропадает дублирование запросов и лишний объём передаваемых данных.
