# InsureTech. Проектная работа 8 спринта

Я выступаю в роли архитектора сервиса InsureTech, страхового агрегатора. В этой работе я прорабатываю масштабирование, отказоустойчивость и интеграции для высоконагруженной realtime-среды.

## Структура репозитория

- `Task1` содержит технологическую архитектуру to-be в draw.io и пояснение к принятым решениям
- `Task2` содержит манифесты Kubernetes (Deployment, Service, HPA), нагрузочный сценарий Locust и подтверждающие логи
- `Task3` содержит разбор проблем текущей архитектуры и обновлённую диаграмму контейнеров C4
- `Task4` содержит диаграмму C4 для продажи ОСАГО с паттернами отказоустойчивости
- `Task5` содержит схему GraphQL для сервиса client-info
- `Task6` содержит конфигурацию Nginx с ограничением количества запросов
